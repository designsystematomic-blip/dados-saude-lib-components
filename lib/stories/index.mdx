# Storybook — Design System (Design System Atomic)

Resumo rápido do Storybook para este design system.

**Conteúdo incluído**

- **Componentes:** 24
- **Ícones:** 23
- **Arquivo de cores:** `lib/styles/colors.css` (variáveis CSS definidas)
- **Tamanhos / fontes:** `lib/styles/fonts.css` (declarações `@font-face` e variáveis de família de fonte)
- **Reset global:** `lib/styles/reset.css`
- **Stories:** cada componente tem seu `.stories.tsx` dentro de `lib/components/<Component>`

---

**Onde estão os arquivos importantes**

- Componentes: `lib/components/` (cada pasta de componente contém `Component.tsx` e `Component.stories.tsx`)
- Ícones: `lib/icons/` (cada ícone em sua pasta, `index.ts` expõe todos)
- Variáveis de cor: `lib/styles/colors.css` — lista de variáveis CSS disponíveis (ex.: `--color-primary`, `--bg-primary`, `--button-bg-primary`, etc.)
- Fontes e famílias: `lib/styles/fonts.css` — declara `@font-face` para `Poppins` e `Outfit` e define variáveis: `--font-family-base`, `--font-family-secondary`, `--font-family-tertiary`.
- Reset global: `lib/styles/reset.css`
- Bundle/style distribuído: `dist/styles/style.css` (após `npm run build`)

---

**Exemplo de uso em um projeto consumidor (aplicar estilos globais)**

Se você instala o pacote via npm e precisa aplicar os estilos globais do design system no seu projeto (incluindo Storybook do consumidor), importe o CSS gerado no ponto de entrada do app (ex.: `src/main.tsx`) ou no `preview` do Storybook:

```ts
// importe no entrypoint do app
import 'design-system-atomic/dist/styles/style.css';

// ou, para Storybook do projeto que consome a lib, em .storybook/preview.ts
import 'design-system-atomic/dist/styles/style.css';
```

Observação: o nome e caminho do CSS podem variar conforme a versão do pacote — verifique `dist/styles/` após `npm run build`.

**Recomendações e notas**

- Garanta que o projeto consumidor importe o CSS da lib (ex.: `dist/styles/style.css`) ou carregue `lib/styles/style.css` durante o build para que as variáveis e resets estejam disponíveis.
- Se o consumidor usa Vite, importe o CSS a partir do `node_modules` (conforme exemplo acima) ou configure o bundler para servir os assets da lib.
- Stories de componentes estão em `lib/components/*/*.stories.tsx` — edite ou adicione stories conforme necessário.
- Add-ons úteis presentes: a11y (ver `package.json` e `.storybook` config).
